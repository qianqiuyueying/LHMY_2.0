# LHMY 2.0 维护性重构（独立规格）

本目录用于记录 **LHMY_2.0** 本次“代码去重/可维护性优化”的独立规格与任务清单。

## 目标

- **不改变现有功能与行为**（包括接口调用语义、跳转/提示、鉴权/会话处理）。
- 收敛 AI 生成造成的重复：重复函数/重复文件/重复依赖/重复封装。
- 将重复点优先收敛到“人类可理解、可追踪、可回滚”的模块边界。

## 非目标

- 不做部署形态升级（不强制 monorepo/workspaces、不改发布流程），除非为了“代码去重”所需的最小构建配置调整。
- 不统一三端 Web 的 UI 组件库（admin: Element Plus；website: Naive UI；h5: Vant）。

## 规格入口

- `requirements.md`：范围、约束、验收口径（DoD）
- `tasks.md`：可勾选任务拆解与证据记录（实施时同步更新）
