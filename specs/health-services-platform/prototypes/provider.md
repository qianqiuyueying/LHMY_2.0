# 健康场所/服务提供方后台—字符原型

## 1. 主框架

```
┌──────────────────────────────────────────────────────────────────────────┐
│ 顶栏：LOGO | 场所后台 | 当前场所[▼] | 通知[ ] | 账号[staff▼]                 │
├───────────────┬──────────────────────────────────────────────────────────┤
│ 侧栏菜单        │ 主内容区                                                     │
│ - 工作台        │                                                              │
│ - 场所信息      │                                                              │
│ - 服务管理      │                                                              │
│ - 排期/容量     │                                                              │
│ - 预约管理      │                                                              │
│ - 核销/记录     │                                                              │
│ - 员工/角色     │                                                              │
└───────────────┴──────────────────────────────────────────────────────────┘
```

## 2. 关键页面原型

### 2.1 工作台（今日预约与待核销）

```
┌──────────────────────────────────────────────────────────────────────────┐
│ 今日：预约待确认 | 今日到店 | 待核销 | 已核销                               │
├──────────────────────────────────────────────────────────────────────────┤
│ 列表A：待人工确认预约（若配置为 MANUAL）                                     │
│  预约ID | 用户 | 服务 | 时间 | 操作[确认][拒绝/取消]                         │
├──────────────────────────────────────────────────────────────────────────┤
│ 列表B：待核销（今日/近期）                                                   │
│  用户 | 权益类型 | 服务 | 预约时间 | 操作[去核销]                            │
└──────────────────────────────────────────────────────────────────────────┘
```

### 2.2 场所信息（对外展示信息）

```
表单：场所名称 | 地址 | 联系电话 | 营业时间 | 城市/区域 | 简介 | 图片/资质
[保存]
```

### 2.3 服务管理（场所可提供的服务类别）

```
Tab：[基建联防上架商品/服务] [健行天下特供服务]

[基建联防上架商品/服务]
列表：名称/类目 | 类型(商品/服务) | 上架状态 | 价格 | 操作[编辑/上架/下架]

[健行天下特供服务]
列表：服务类目(serviceType) | 标题(title) | 是否需要预约(bookingRequired) | 核销方式(redemptionMethod) | 适用区域(applicableRegions) | 状态 | 操作[编辑/停用]
编辑弹窗：服务类目 | 标题 | 说明 | 是否需要预约 | 核销方式 | 适用区域标签 | 备注
```

### 2.4 排期/容量（预约资源配置）

```
筛选：服务类别[▼] 日期范围[起-止]
┌──────────────────────────────────────────────────────────────────────────┐
│ 日历/表格：日期 | 时段(09:00-10:00...) | 总容量(capacity) | 剩余容量(remainingCapacity) | 操作[调整] │
└──────────────────────────────────────────────────────────────────────────┘

说明（最小可执行口径）：
- 时段格式统一为 `HH:mm-HH:mm`（24小时制，例如 `09:00-10:00`），用于与预约取消窗口等规则一致判定。
```

### 2.5 预约管理（确认/改期/取消）

```
筛选：状态[待确认/已确认/已取消/已完成] 日期[起-止] 服务类别[▼] 关键词[____]
┌──────────────────────────────────────────────────────────────────────────┐
│ 预约ID | 用户 | 服务 | 日期/时段 | 状态 | 确认方式(AUTO/MANUAL) | 操作       │
│        |      |      |           |      |                      | [确认/取消] │
└──────────────────────────────────────────────────────────────────────────┘

预约详情：
基本信息 | 关联权益 | 用户备注 | 场所备注 | 操作日志
```

### 2.6 核销（扫码/输入券码）

> 对应设计文档正确性属性：核销成功才扣次数、需要预约的服务先预约后核销等。

```
┌──────────────────────────────────────────────────────────────────────────┐
│ 核销方式： (●) 扫码(二维码)   ( ) 输入券码                                  │
│                                                                              │
│ [扫码区域/相机调用占位]                                                      │
│ 或 券码输入： [____________________]  [校验]                                 │
├──────────────────────────────────────────────────────────────────────────┤
│ 校验结果：                                                                  │
│  - 用户：xxx   权益：服务类别/剩余次数/有效期                                │
│  - 是否需要预约：是/否   预约状态：已确认/无                                 │
│  - 可核销：是/否（若否显示原因）                                             │
│ [确认核销]                                                                  │
└──────────────────────────────────────────────────────────────────────────┘
```

### 2.7 核销记录

```
筛选：日期[起-止] 服务类别[▼] 状态[成功/失败] 操作员[▼]
表格：核销时间 | 用户 | 权益ID | 服务 | 方法(扫码/券码) | 结果 | 失败原因 | 详情
```
