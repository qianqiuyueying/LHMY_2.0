### 2.11 经销商入驻状态机（平台侧）

```
状态枚举（示例）：
- DRAFT（待提交）：经销商填报中，未提交
- SUBMITTED（待审核）：已提交，等待平台审核
- RETURNED（退回补充）：资料不全，退回补充后可再次提交
- REJECTED（已驳回）：审核不通过，流程结束（可重新发起新申请）
- APPROVED（已通过待启用）：审核通过，待平台启用
- ACTIVE（已启用）：可正常产生归属订单/结算
- SUSPENDED（已停用）：平台停用（监管/风控/违规），不可继续产生新归属

状态转移：
DRAFT -> SUBMITTED -> (APPROVED | RETURNED | REJECTED)
RETURNED -> SUBMITTED
APPROVED -> ACTIVE
ACTIVE -> SUSPENDED
SUSPENDED -> ACTIVE  （如复审通过/解除停用）
```

平台动作（admin）：
- 审核：通过/驳回/退回补充（记录原因与证据）
- 启用：将 APPROVED 置为 ACTIVE
- 停用/启用恢复：ACTIVE<->SUSPENDED（记录原因、影响范围、是否影响结算/链接）

### 2.11A 经销商停用/冻结影响矩阵（最小可执行口径）

> 说明：此处的“停用/冻结”对应状态 `SUSPENDED`。目标是给运营/风控一个**最小可落地**的系统行为：冻结增量、保证存量可控、结算可对账。

```
对象：经销商（dealer）
状态：ACTIVE / SUSPENDED

┌──────────────────────────────┬──────────────┬──────────────────────────────────────────────┐
│ 场景                         │ ACTIVE       │ SUSPENDED（停用/冻结）                        │
├──────────────────────────────┼──────────────┼──────────────────────────────────────────────┤
│ 停用后是否禁止新订单         │ 否           │ 是：禁止产生“归属到该经销商”的新健行天下购买订单 │
│                              │              │（链接/参数仍可访问，但下单需被拦截或归属失败）   │
├──────────────────────────────┼──────────────┼──────────────────────────────────────────────┤
│ 存量订单是否仍可履约/结算     │ 是           │ 是：不影响已支付存量订单的权益交付与使用；       │
│                              │              │结算进入“冻结结算/需平台审核放行”模式（见下）      │
├──────────────────────────────┼──────────────┼──────────────────────────────────────────────┤
│ 权益发放是否受影响           │ 否           │ 默认否：已支付订单的权益仍正常发放；             │
│                              │              │是：已支付订单的权益发放进入“人工审核后发放”模式    │
│                              │              │（用于风控/合规；避免停用后仍自动交付带来风险）     │
├──────────────────────────────┼──────────────┼──────────────────────────────────────────────┤
│ 对已有结算周期的影响         │ 正常结算     │ 不回滚已结算周期；                               │
│                              │              │未结算周期：冻结出账/打款，需平台复核后放行或作废  │
└──────────────────────────────┴──────────────┴──────────────────────────────────────────────┘
```

admin 最小动作（对应矩阵落地）：
- 停用时：同时停用该经销商“链接/参数”投放能力（或标记为无效），并冻结其未结算周期出账
- 停用时：将该经销商归属的“已支付待发放权益”置为人工审核队列（通过后才发放）
- 恢复时：允许新归属订单 + 解除未结算周期冻结（可选择要求复审）

### 2.13 服务提供方入驻状态机（平台侧）

```
状态枚举（示例）：
- V_DRAFT（待提交）
- V_SUBMITTED（待审核）
- V_RETURNED（退回补充）
- V_REJECTED（已驳回）
- V_APPROVED（已通过待启用）
- V_ACTIVE（已启用）
- V_SUSPENDED（已停用）

状态转移：
V_DRAFT -> V_SUBMITTED -> (V_APPROVED | V_RETURNED | V_REJECTED)
V_RETURNED -> V_SUBMITTED
V_APPROVED -> V_ACTIVE
V_ACTIVE -> V_SUSPENDED
V_SUSPENDED -> V_ACTIVE
```

平台动作（admin）：
- 审核：通过/驳回/退回补充（记录原因与证据）
- 启用：将 V_APPROVED 置为 V_ACTIVE，允许其在平台侧被选择为“供给主体”
- 停用/恢复：V_ACTIVE<->V_SUSPENDED（联动影响：电商上架、健行天下特供服务可用性）

### 2.13A 服务提供方停用/冻结影响矩阵（最小可执行口径）

> 说明：服务提供方停用/冻结对应状态 `V_SUSPENDED`。平台型电商下，商品与履约责任主体为商家，但平台需要冻结增量风险并维持存量可控。

```
对象：服务提供方（商家/场所）
状态：V_ACTIVE / V_SUSPENDED

┌──────────────────────────────┬──────────────┬────────────────────────────────────────────────────┐
│ 场景                         │ V_ACTIVE     │ V_SUSPENDED（停用/冻结）                            │
├──────────────────────────────┼──────────────┼────────────────────────────────────────────────────┤

│                              │              │健行天下特供服务不可被兑换（不可核销/不可新预约）        │
├──────────────────────────────┼──────────────┼────────────────────────────────────────────────────┤
│ 存量订单是否仍可履约/结算     │ 是           │ 最小口径：允许存量订单履约，但进入“平台监管模式”：      │
│                              │              │最小口径：存量订单默认优先“退款/改派”，由平台介入协调：    │
│                              │              │- 若已有预约：默认取消并发起退款/改派（必要时可例外放行履约）│
│                              │              │- 结算：未结算部分冻结，待平台复核（退款/仲裁结果）后放行   │
├──────────────────────────────┼──────────────┼────────────────────────────────────────────────────┤
│ 权益发放是否受影响           │ 否           │ 电商订单权益/发货：不应新增（因禁止新订单）；             │
```

admin 最小动作（对应矩阵落地）：
- 停用时：一键下架/禁售该商家电商商品；禁用其健行天下特供服务供给；冻结未结算周期
- 恢复时：解除禁售/恢复供给（可要求复审），并对冻结结算做放行/调整

## 3. 关键说明（与规格一致性）

- “企业信息库/绑定关系”的详细审核流（是否必审、状态枚举等）若需要，可在后续规格中补充；本原型仅描述查询与维护入口。

- 经销商除自用后台外，平台运营后台（admin）提供“监管/配置/对账”入口（经销商信息、层级、链接参数、结算记录等）。

- 基建联防定位为“平台型电商（多服务提供方入驻）”：平台不直接作为商品售卖主体；商品、订单、履约与售后首先以商家为责任主体；admin 侧以监管、审核与仲裁为主。

- 第三方入驻（经销商/服务提供方）采用“申请单”驱动：申请—审核—启用—监管—停用；具体状态机见对应章节。
