# 小程序端（健康主题电商平台：三业务线统一入口与使用端）—字符原型

> 说明：本原型聚焦“信息结构 + 关键流程页面”，小程序内包含基建联防（电商）、健行天下（权益使用）、职健行动（企业绑定/员工身份）。健行天下的购买入口在 H5，本小程序不提供购买页。

> 补充：小程序支持“可配置页面”，由 admin 配置首页快捷入口/运营位跳转到 **聚合页/信息页/WebView/固定路由**。聚合页用于商家/产品聚合展示（一次聚合一种对象），支持侧边栏与市/省/国维度。

## 1. 全局导航（示例）

```
底部Tab： [首页]  [商城]  [权益]  [订单]  [我的]
```

## 2. 页面原型

### 2.1C AI 对话入口（首页右下角 AI 旋钮）

```
规则：

- 会话不持久化（退出即清空）
```

### 2.1A 聚合页（可配置：场所聚合/产品聚合，支持侧边栏）

> 入口示例：快捷入口“场所” → 场所聚合页（按地区维度聚合） → 场所详情。

场所聚合页（一次聚合一种对象：场所/门店）：
```
┌───────────────────────────┐
│ 顶部：标题[场所] 搜索[____]  │
├──────────┬────────────────┤
│ 侧边栏    │ 内容区           │
│ 维度：市  │ 二级标签(可选)： │
│ - A市     │ [全部][按摩][游泳]│
│ - B市     │                │
│ - C市     │ 场所卡片列表：   │
│ （维度可切：市/省/国，默认由 admin 配置；地区筛选依据为“已审核的场所归属地区字段”）│
│          │ [场所A] 区域Tag  │
│          │ [场所B] ...      │
└──────────┴────────────────┘
```

产品/服务聚合页（一次聚合一种对象：产品/服务）：
```
┌───────────────────────────┐
│ 顶部：标题[产品] 搜索[____]  │
├──────────┬────────────────┤
│ 侧边栏    │ 内容区           │
│ 维度：省  │ 二级标签(可选)： │
│ - A省     │ [全部][健身][理疗]│
│ - B省     │                │
│          │ 卡片列表：        │
│          │ [产品/服务1] 价/标签│
│          │ [产品/服务2] ...  │
└──────────┴────────────────┘
```

### 2.5 预约（若服务需要预约）

```
┌───────────────────────────┐
│ 选择场所                      │
│ 筛选：区域[▼] 服务[▼]        │
├───────────────────────────┤
│ 场所列表（可用/不可用原因）   │
│ [场所A] 距离/地址 [可预约]    │
│ [场所B] ...      [不可用:限制]│
└───────────────────────────┘

┌───────────────────────────┐
│ 选择日期/时段                 │
│ 日历 + 时段列表(可用容量)     │
│ [提交预约]                   │
└───────────────────────────┘

说明（最小可执行口径）：
- 时段格式统一为 `HH:mm-HH:mm`（24小时制，例如 `09:00-10:00`），用于与预约取消窗口等规则一致判定。

预约结果：
┌───────────────────────────┐
│ 状态：待确认/已确认（AUTO/MANUAL） │
│ 预约信息：场所/时间/服务类别   │
│ [取消/改期]（规则限制）        │
└───────────────────────────┘
```

企业绑定页（职健行动）：
```
┌───────────────────────────┐
│ 企业绑定                     │
│ 输入企业名称：[__________]    │
│ 匹配建议：                   │
│  - 企业A                     │
│  - 企业B                     │
│ [确认绑定]                   │
└───────────────────────────┘
```

说明（最小可执行口径）：
- 提交后进入“待审核”状态；仅当平台运营后台（admin）审核通过后，用户才获得“员工身份（EMPLOYEE）”并享受员工价/企业折扣。

### 2.8 场所详情（服务提供方展示，面向消费者）

> 数据来源：服务提供方在其后台维护资料，提交后经 admin 审核发布；小程序仅展示“已发布”版本。

```
┌───────────────────────────┐
│ 场所/品牌详情                │
├───────────────────────────┤
│ 头图/Logo  场所名  区域标签    │

├───────────────────────────┤

│ - 服务B                       │
├───────────────────────────┤

│ [去商城] [去预约(如适用)] [导航]│
└───────────────────────────┘
```
